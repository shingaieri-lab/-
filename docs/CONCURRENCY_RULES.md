# 同時実行・競合状態の防止ルール

## トークンリフレッシュ

- リフレッシュ中フラグを設け、進行中の場合は他のリクエストを待機させる
- 複数リクエストが同時にリフレッシュを呼ばないようにする

## データ更新の競合

- 「全件読み込み → 加工 → 全件上書き」は競合状態（Last-Write-Wins）が発生する
- ロック機構またはアトミック操作を使う

## レート制限

- バックエンド（KV等）が障害のときは安全側に倒す（429を返す）
- ❌ catch { next(); }（障害時は素通り）
- ✅ catch { res.status(429).json({ error: 'Rate limit unavailable' }); }

## 重要処理の安全対策

- パスワードリセット等ではレスポンスのステータスを先にチェックする
- 非JSON応答にも .catch(() => ({})) で対応する
