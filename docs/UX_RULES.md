# UX・アクセシビリティルール

## タッチ操作・視認性

- ボタン・タッチターゲットは最低 44×44px。padding: "3px 10px" のような小さいボタン禁止
- ボタンやアイコンの opacity は常時 0.7 以上。ホバー時のみ表示はタッチデバイスで使えない
- 固定幅（width: 380px 等）はモバイルで圧迫する。max-width や % でレスポンシブにする

## 通知・フィードバック

- 成功/エラーメッセージは最低5秒表示。または ✕ ボタンで手動クローズ
- バルク操作の完了時はトースト通知で結果を表示する（例：「10件追加しました」）
- すべての非同期処理で統一されたローディング表示（スピナー）を使う

## 導線

- 処理完了後は次のアクション（メール送信・候補日検索等）への導線を表示する
- 削除確認に window.confirm を使わない。アプリ内モーダルで確認を取る

## エラーメッセージ

- ユーザー向けメッセージに技術用語を含めない（CORS、500、JSON parse error 等）
- 「接続できませんでした。管理者にお問い合わせください」のように平易に書く
